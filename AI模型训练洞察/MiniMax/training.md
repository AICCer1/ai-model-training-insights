# MiniMax 系列模型训练洞察

## 模型系列

- **MiniMax-abab** (2023年) - 首代模型
- **MiniMax-abab5** (2024年) - 显著提升
- **MiniMax-abab5.5** (2024年) - 优化版本
- **MiniMax-abab6** (2024年中期) - 性能大幅提升
- **MiniMax-abab6.5s** (2024年) - 快速版本
- **MiniMax-M2** (2024年) - 第二代模型
- **MiniMax-M2.5** (2025年) - 最新旗舰

## 训练流程

### 1. 预训练 (Pre-training)

#### 数据来源与配比

| 版本 | 预训练 Token 数 | 数据来源 |
|------|----------------|----------|
| MiniMax-abab5 | ~2T | 网页、代码、书籍 |
| MiniMax-abab6 | ~3T+ | 大规模高质量数据 |
| MiniMax-M2 | ~4T+ | 多源大规模数据 |
| MiniMax-M2.5 | >5T | 极大规模高质量数据 |

**数据配比策略**：
- **英文网页**：25%
- **中文网页**：25%
- **代码**：20%
- **学术/论文**：15%
- **图书**：10%
- **其他**：5%

#### 数据清洗方法

1. **质量过滤**：
   - 多级质量评估
   - 语言识别与分类
   - 内容安全过滤

2. **去重**：
   - 精确去重
   - 近似去重

3. **分词器**：
   - **BPE分词**
   - **词表大小**：约200k (较大词表)

#### 训练配置

| 模型 | 隐藏层 | 注意力头 | FFN隐藏 | 上下文长度 |
|------|--------|----------|---------|-----------|
| abab6 12B | 40 | 32 | 13696 | 32K/128K |
| abab6 70B | 80 | 64 | 28672 | 128K |
| M2 12B | 40 | 32 | 13696 | 32K/128K |
| M2 456B | 80 | 64 | 245760 | 256K |
| M2.5 16B | 48 | 40 | 16384 | 32K/200K |
| M2.5 620B | 96 | 80 | 327680 | 1M (理论) |

**训练超参数**：
- **优化器**：AdamW
- **学习率**：峰值 2e-4 ~ 3e-4
- **学习率调度**：Cosine Annealing
- **权重衰减**：0.1
- **梯度裁剪**：1.0

### 2. 有监督微调 (SFT)

#### 数据来源

| 数据集类型 | 描述 |
|-----------|------|
| 人工标注对话 | 高质量指令-回复对 |
| 开源SFT数据 | 精选开源数据 |
| 合成数据 | 模型生成+筛选 |

#### 训练配置
- **学习率**：1e-5 ~ 2e-5
- **Epochs**：2-3
- **批量大小**：动态

### 3. 对齐训练 (RLHF / GRPO)

#### 对齐方法
- **奖励模型**：基于人类偏好训练
- **PPO/GRPO**：策略优化
- **DPO**：直接偏好优化

## 架构特点

### 1. 标准Transformer架构

- **Decoder-only**
- **RoPE位置编码**
- **SwiGLU激活函数**

### 2. 注意力优化

- **Multi-head Attention**
- **Grouped Query Attention**
- **Flash Attention**

### 3. 长上下文支持

- **YaRN**：动态rope缩放
- **1M上下文**：M2.5支持极长上下文

### 4. 训练优化

- **Mixed Precision**
- **Gradient Checkpointing**
- **分布式训练**

## MiniMax-M2.5 特性

1. **极长上下文**
   - 理论支持1M上下文
   - 实际应用200K

2. **多语言支持**
   - 优化中文/英文
   - 支持多语言场景

3. **Function Calling**
   - 工具调用能力
   - 自动化任务

4. **快速响应**
   - **abab6.5s**：快速版本
   - 优化推理速度

## 数据格式

### 预训练格式
```json
{"text": "文本内容", "source": "web|code|book"}
```

### SFT格式
```json
{
  "messages": [
    {"role": "user", "content": "问题"},
    {"role": "assistant", "content": "回答"}
  ]
}
```

### DPO格式
```json
{
  "prompt": "问题",
  "chosen": "好的回答",
  "rejected": "差的回答"
}
```

## 平台化公开线索

### MiniMax 开放平台
MiniMax 公开出来更多的是：
- 模型能力
- API 平台
- 企业接入

相比云厂商，它公开的底层训练平台细节更少。

### 对这个仓库的意义
MiniMax 这条线更适合观察：
- 模型公司如何把训练成果服务化
- 为什么很多模型公司公开的是开放平台与部署能力，而不是底层训练集群细节
